### 进程

1. CUP工作原理

```
1. cpu是一个信息一个任务进行处理的，对于用户感知而言，之所以能同时处理多个任务，因为速度很快；
2. 处理多个任务的逻辑为，不断在任务间轮训，直到所有的任务结束或者中途退出；
```

2. 进程切换

```
1. 进程是cpu加载程序指令在一堆数据上的一次执行，计算机上有很多进程，需要cpu在对歌进程间来回切换，轮训处理，这就是进程切换；
2. cpu只会处理就绪态的进程，也就是所有准备工作都已完成，只需要cpu的时候；
3. cup处理的进程，此进程会临时占有cpu；
4. 如果执行中的进程，发生意外终止，会把进程踢出轮训池，进程状态变更为阻塞态；
```

3. 进程间通信

```
1. 进程是相互独立的，有自己独立的存储空间，进程间数据不互通不共享；
2. 为了进程间相互通讯，实现的方式：
    共享存储区：
        在进程之外，开辟共有的内存空间，都可以进行读写，前提条件是A进程处理完成前，B无法进入共享存储处理；
    消息传递：
        系统给每个进程分配了一个存储消息的空间；
        一个消息包括两部分：消息头和消息体；
    管道通信：
        本质是一个共享文件，pipe文件，是系统开辟的一块特殊的缓冲空间，系统提供了一套管道创建和销毁的方法；
        进程间有互斥性，当A进程写满或者读满之后，B进程才可以进行处理；
        管道数据从管道中读取之后就没了，不可复读，一次性；
        管道分为有名和无名，父子进程间通信一般使用无名；
```
